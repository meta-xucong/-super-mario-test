# 玩法调参与数值验证

状态：流程基线；数值待冻结

## 调参纪律

只允许通过版本化的 `tuning` 数据调整物理；禁止散落魔法数字。每个参数有单位、所有者、参考场景、允许范围和 revision。一次提交只能改变一个假设：例如“空中反向过强”，不能同时改加速度、重力和跳跃初速度。

## 初始参数表模板

| 参数 | 单位 | 测量场景 | 验收指标 |
| --- | --- | --- | --- |
| `walkMaxSpeed` | tile / s | 平地从静止到稳定行走 | 固定距离耗时 |
| `runMaxSpeed` | tile / s | 平地持续冲刺 | 固定距离耗时、无穿透 |
| `groundAccel` | tile / s² | 静止起跑 | 到达 90% 速度 tick 数 |
| `airAccel` | tile / s² | 跳跃中反向 | 修正能力不超过参考容限 |
| `friction` / `skidDecel` | tile / s² | 松键、反向 | 停止距离和滑行帧数 |
| `gravity` | tile / s² | 自由下落 | 固定高度落地 tick 数 |
| `jumpVelocity` | tile / s | 满按跳跃 | 顶点高度、总滞空时间 |
| `jumpHoldWindow` | tick | 点按 / 长按 | 两种顶点高度差 |
| `coyoteTicks` | tick | 离边缘起跳 | 容错不改变必经跳跃 |
| `jumpBufferTicks` | tick | 落地前按跳 | 接地后触发一致 |
| `stompBounceVelocity` | tile / s | 踩踏敌人 | 反弹高度和状态切换 |

## 基准场景

建立命名输入录像：`run-flat`、`short-jump`、`long-jump`、`reverse-skid`、`stomp`、`head-hit`、`pipe-entry`、`goal-top`、`goal-bottom`。每段录像输出每 30 tick 的快照哈希，以及关键帧的 x、y、速度、状态、接地标志与事件序列。

## 容限规则

布局冻结后，地面位置、碰撞触发、敌人出现顺序、方块内容和终点流程必须完全一致。运动数值允许在基准录像上有小幅位置 / 时间容限，但不得导致原本可达路径失败、原本不可达路径成立，或改变交互先后。任何容限值由玩法负责人明确写入测试，不允许“看起来差不多”。

## 调参验收顺序

先校准平地速度与惯性，再校准跳高 / 滞空，接着校准空中控制、踩踏反弹、顶砖和伤害。最后才检查镜头前瞻与动画节奏。镜头、动画和 VFX 无权掩盖物理误差。
