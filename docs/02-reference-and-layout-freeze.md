# 参考取证、布局清单与冻结流程

状态：流程基线；具体布局数据待冻结

## 原则

关卡的“完全贴合”必须是可测试的工程结论，而不是视觉印象。只使用有权访问的游玩录像、公开可观看画面或自行合法录制的操作结果作为观察材料；本仓库只保存标注坐标、描述和校验摘要，不保存受保护的原始媒体或提取数据。

## 参考清单与证据链

在 `docs/reference-register.md`（开工时创建）记录每项材料：`referenceId`、来源描述、访问日期、用途、可见区间、记录者、复核者、是否允许仓库存放。每个布局对象必须至少关联一个 `referenceId` 和画面区间。若两个材料冲突，优先采用可重复操作得到的结果，并记录裁决。

## 标注流程

1. 建立逻辑网格：确认 tile 尺寸、地面基准行、屏幕宽度与坐标原点。
2. 逐屏编号：每次相机滚动产生固定编号，并记录相邻屏重叠部分。
3. 记录静态物：地形、砖、问号砖、隐藏块、管道、旗杆、城堡、坑边。
4. 记录动态物：敌人出生点、初始朝向、移动边界、道具内容与硬币。
5. 记录脚本：入管条件、转场、出口锚点、终点触发、时间换分和动画锁定。
6. 复核者在无提示条件下按材料重建第二份清单；差异逐项裁决后才能合并。

## 布局清单最低字段

| 分类 | 必填字段 |
| --- | --- |
| 关卡 | `levelId`、`layoutRevision`、网格单位、边界、出生点、终点 ID |
| Tile | `tileId`、层、x、y、碰撞类型、参考 ID |
| 实体 | `entityId`、类型、x、y、朝向、初始状态、参考 ID |
| 触发器 | `triggerId`、形状、条件、结果、优先级、参考 ID |
| 奖励区 | `areaId`、入口、出口、返回锚点、持久化规则 |
| 复核 | 记录者、复核者、日期、差异记录、批准人 |

ID 不可复用，例如 `l11-ground-004`、`l11-walker-003`、`l11-pipe-002`。坐标使用逻辑格，禁止根据高清贴图边缘手工偏移碰撞体。

## 可达性与一致性检查

冻结前必须生成无美术碰撞图，并人工确认：正常助跑可越过每个必经缺口；玩家不会出生在实心格中；奖励区只有预期入口和出口；所有砖块内容一致；每个终点触发器唯一；任何隐藏物都按原规则触发而非被升级特效泄露。

## 冻结、解冻与版本号

冻结包包含布局 JSON、实体清单、触发器清单、碰撞图、差异表和复核签字。冻结后只允许增加不影响逻辑的视觉注释。需要改动时创建新 revision，列出每个 ID 的新增 / 删除 / 移动和理由；旧 replay 必须保留并标明对应版本。未经双人复核的“临时修正”不允许进入主分支。
