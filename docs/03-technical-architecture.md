# 技术架构契约

状态：基线

## 架构目标

使用 TypeScript 构建浏览器游戏：模拟确定、数据驱动、渲染可替换、资源可追踪、回放可验证。渲染引擎只负责画面；它不得定义角色运动、碰撞、分数或关卡规则。

## 运行循环

```text
浏览器输入 → 动作采样 → 固定步模拟（1/120 秒）→ 事件队列 → 快照
                                                  ├→ 音频导演
                                                  ├→ UI 状态
                                                  └→ 渲染插值 → WebGL 画面
```

- 输入按渲染帧采样，边沿事件保留到下一个固定步消费。
- 模拟使用固定 tick、确定性排序和显式种子；禁止在模拟层读取当前时间、DOM 或随机数。
- 渲染仅对前后快照插值；绝不把插值位置写回模拟状态。
- 单帧补算设置上限并记录 `simulationOverrun`，不可通过增大 `dt` 追赶。

## 模块与禁止依赖

| 模块 | 负责 | 不得依赖 |
| --- | --- | --- |
| `core` | 时钟、ID、事件、序列化、种子 | UI、渲染器、浏览器 API |
| `input` | 设备采样、动作映射、重绑 | 玩法实体 |
| `simulation` | 玩家、物理、关卡、规则 | 图形 / 音频实现 |
| `render` | 相机、图层、动画表现、VFX | 改写模拟状态 |
| `audio` | 事件到混音 / 音效 | 改写分数或实体 |
| `ui` | HUD、菜单、设置 | 直接移动玩家 |
| `infra` | 资源、日志、持久化、平台适配 | 玩法判定 |

## 状态、事件与排序

模拟快照最少包含：tick、关卡 revision、玩家状态、动态实体、已触发块 / 硬币、剩余时间、分数、生命、相机逻辑锚点和随机种子。事件使用不可变对象：`tick`、`sequence`、`type`、`sourceId`、`targetId`、`payload`。同 tick 按优先级、`sourceId`、`targetId`、`sequence` 排序，确保回放跨设备一致。

事件类型分组：`InputConsumed`、`BlockHit`、`ItemSpawned`、`CoinCollected`、`EnemyStomped`、`PlayerDamaged`、`PlayerDied`、`PipeEntered`、`GoalTouched`、`ScoreChanged`、`TimeConverted`、`SceneChanged`。一个事件只有一个状态所有者；例如 `CoinCollected` 由规则层生成分数变化，UI 只显示。

## 数据格式与兼容性

关卡、调参、输入配置和资产清单都带 `schemaVersion`、`revision`、稳定 ID。解析失败必须显式阻断加载并报告字段路径，不能套默认值偷偷继续。存档和 replay 必须记录对应的关卡 / 调参 revision；版本不一致时只可提示不兼容，不可猜测迁移。

## 性能与可观测性

开发叠层显示 FPS、模拟 tick、补算次数、实体数、碰撞盒、draw call、纹理内存和最近事件。正式环境只收集必要的匿名崩溃与性能摘要。目标是 60 FPS 渲染、稳定 120 Hz 模拟；性能降级顺序为后处理 → 粒子 → 远景动画，绝不降级物理和输入。

## 安全与恢复

所有外部资源使用固定清单和内容哈希；加载失败显示可重试界面。焦点丢失立即暂停，恢复时清空键位边沿以防“自动跳”。本地设置仅保存键位、音量、无障碍选项和练习进度，不存储不必要的个人信息。
