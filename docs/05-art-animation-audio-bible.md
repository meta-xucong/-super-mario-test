# 美术、动画与音频圣经

状态：基线

## 创作方向

视觉主题为“晴空下可触摸的微缩探险世界”。表达目标是层次、材质、动势和光照的现代化，而非复刻原作的角色比例、像素轮廓、砖块纹样、字体、配色排列或音乐彩蛋。任何资产在缩略图下都应和原作角色 / 素材明显可区分。

## 可读性规则

- 玩家、敌人、危险边缘、可顶击物和可收集物必须在明度、色相或轮廓中至少有两项差异。
- 地形装饰不能改变逻辑边界的视觉认知；碰撞边沿应由清晰轮廓、阴影或材质分割提示。
- 前景、景深、粒子和屏幕震动不能遮挡角色脚部、落点、敌人或奖励物。
- 隐藏内容不使用发光、脉冲、箭头或不符合参考的镜头提示泄露答案。

## 资产包与命名

| 包 | 最低交付 | 命名例 |
| --- | --- | --- |
| 角色 | 全状态动画、受伤 / 死亡、碰撞参考 | `hero_run_01` |
| 敌人 | 巡逻、踩踏、受击、消失 | `walker_stomp_01` |
| 地形 | 地面、边缘、砖、管道、旗帜、城堡 | `terrain_pipe_tall_a` |
| 环境 | 天空、远景、植被、地下区 | `bg_hills_far_02` |

| 音频 | 音乐分段、动作 SFX、环境声 | `sfx_block_hit_stone_01` |

每项资产登记作者、工具、来源、授权、生成式工具使用情况、导出日期、内容哈希和审核状态。源文件与运行时导出文件分开保存。

## 动画与特效

动画由游戏状态驱动而非反向控制状态。跑步、急停、起跳、上升、下落、顶砖、踩踏、受伤、进管、触旗均需定义进入条件、可打断性和事件标记。粒子必须对象池化，按低 / 中 / 高质量有明确上限；VFX 只读取事件和位置，不能产生伤害或收集判定。

## 音频

音乐、主题动机、和声、节奏、音色设计均为原创。混音划分音乐、动作、环境、UI、旁白 / 字幕五条总线；相同硬币或碎屑事件需要限频和合并。提供音乐 / 效果分音量、动态范围、静音和音效字幕。完成转场时先锁定游戏状态，再按事件驱动音频与画面，避免跳帧重复播放。

## 评审清单

- [ ] 资产不含受保护名称、Logo、提取或近似复制的图音素材。
- [ ] 任意游戏关键帧中，落点和危险物清晰可见。
- [ ] 4K、1080p、低分辨率与色弱模式下 UI 和互动元素可辨。
- [ ] 资源加载、图集、循环音和销毁路径已纳入性能审查。
