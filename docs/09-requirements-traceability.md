# 需求追踪矩阵

状态：基线

| --- | --- | --- | --- | --- |
| R-01 | 从标题到结算完整可游玩 | 主规格 1、4.4；UX | E2E 全流程测试 | P2 |
| R-02 | 关卡布局和事件顺序可复核 | 参考 / 布局冻结文档 | 逐屏比对、`diff-layout` | P2 |

| R-05 | 管道奖励区和旗杆流程正确 | 主规格 4 | 回放、状态快照、人工通关 | P2 |
| R-06 | 视觉与音频原创且可读 | 美术音频圣经；合规文档 | 资产登记、可读性评审 | P3 |

## 使用规则

- 需求 ID 不因文本润色而改变；被废弃时标为废弃并说明替代项。
- 一个测试失败会使对应需求回到未验证状态。
- P2 之前不允许“用美术完成度”替代玩法 / 数据验证；P4 之前不允许“可玩”替代合规验证。
