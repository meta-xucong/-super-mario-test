# UX、界面与无障碍规格

状态：基线

## 用户流程

```text
启动 → 标题 → 开始正式模式 / 练习模式 / 设置 → 关卡 → 暂停
                                                   ├→ 重新开始
                                                   ├→ 设置
                                                   └→ 返回标题
完成 → 时间结算 → 结果页 → 再玩 / 返回标题
失败 → 死亡动画 → 重开 / 返回标题
```

正式模式保留完整关卡流程和规则。练习模式可从批准的分段锚点开始，但必须清楚标记为非正式成绩，且不能混入正式时间或分数。

## HUD 与界面规则

HUD 显示分数、收集数、关卡标识、剩余时间和生命；所有文字、图标、字体与布局原创。HUD 不覆盖玩家、旗杆、关键砖块或入管入口。暂停是即时、无动画延迟的硬暂停；重新获得焦点后需要一次明确输入才恢复。

## 可访问性最低要求

| 能力 | 要求 |
| --- | --- |
| 操作 | 所有动作可重绑；键盘和手柄全程可用；手柄断连有提示。 |
| 视觉 | 高对比度交互轮廓、色弱配色、可关闭屏闪和镜头震动、可减少粒子。 |
| 听觉 | 独立音量、音效字幕、非声音单独传达关键信息。 |
| 认知 | 清晰焦点态、设置即时预览、简短可撤销提示、练习模式。 |
| 运动 | 允许降低动态背景与后处理；不以快速连按作为唯一操作。 |

## 输入规范

动作名称固定为 `moveLeft`、`moveRight`、`run`、`jump`、`pause`、`confirm`、`back`。设置页面显示冲突键位并要求用户解决；禁止把浏览器保留快捷键静默绑定为唯一操作。触屏控制若后续实施，必须与同一动作映射共享逻辑，不得为触屏建立不同物理规则。

## UX 验收

在无音频、高对比度、减少动态效果、仅键盘、仅手柄五种环境下完成从标题到结算的完整流程。出现任何无法操作、信息只靠颜色 / 声音传达、焦点丢失或暂停后误输入的问题，都按高优先级缺陷处理。
